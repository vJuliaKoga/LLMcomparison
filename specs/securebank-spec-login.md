# SecureBank デモシステム テスト仕様書（ログイン機能抜粋）

## 1. 対象

ログイン機能（基本認証、ログイン試行制限、多要素認証、監査ログ記録）

## 2. テスト対象アカウント

| 役割     | ユーザーID | パスワード |
| -------- | ---------- | ---------- |
| 一般顧客 | customer01 | pass123    |
| 管理者   | admin01    | admin123   |
| 監査者   | auditor01  | audit123   |

## 3. 機能仕様

### 3.1 基本認証

- 入力項目: ユーザーID、パスワード、役割選択
- 検証項目:
  - ユーザーIDとパスワードの一致
  - 選択した役割とアカウント役割の一致
  - 空欄チェック

### 3.2 ログイン試行制限

- 制限回数: 3回まで
- ロック時間: 30秒
- 動作:
  - 3回失敗後、30秒間すべてのログイン試行を拒否
  - ロック中のメッセージ: "🔒 アカウントがロックされています。残りX秒"

### 3.3 多要素認証（MFA）

- タイミング: 基本認証成功後
- 認証コード: 6桁の数字（テスト用: 123456）
- 失敗時: エラーメッセージ表示、ログイン不可

### 3.4 監査ログ記録

- ログイン成功/失敗
- MFA成功/失敗
- ログイン試行回数超過

## 4. エラーハンドリング

### 4.1 主要エラーメッセージ

| エラー分類       | メッセージ                                 | トリガー条件        |
| ---------------- | ------------------------------------------ | ------------------- |
| 入力不足         | "ユーザーIDとパスワードを入力してください" | ログイン時、空欄    |
| 認証失敗         | "認証失敗（残りX回）"                      | ID/パスワード不一致 |
| アカウントロック | "アカウントがロックされています。残りX秒"  | 3回失敗後           |
| MFA失敗          | "認証コードが正しくありません"             | MFA不一致           |

## 5. セキュリティ

- パスワードは入力時に type="password" でマスク
- すべての認証失敗を監査ログに記録

## 6. テスト観点

- 正常系: 正しい資格情報 + 正しいMFAでログイン成功
- 異常系: 認証失敗、MFA失敗
- 境界値: 失敗3回目でロック
- セキュリティ: ロック中再試行、役割不一致
