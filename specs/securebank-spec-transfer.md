# SecureBank デモシステム テスト仕様書（振込機能抜粋）

## 1. 対象

振込機能（入力、バリデーション、承認フロー、成功時処理、監査ログ）

## 2. 前提

- ログイン済みユーザーが振込タブを操作する
- 一般顧客、管理者、監査者の口座情報が存在する

## 3. 入力項目

- 振込先口座番号: 10桁の数字のみ
- 振込金額: 正の整数
- メモ: 任意（文字列）

## 4. バリデーションルール

| 検証項目     | 条件            | エラーメッセージ                               |
| ------------ | --------------- | ---------------------------------------------- |
| 口座番号形式 | 10桁の数字      | "振込先口座番号は10桁の数字で入力してください" |
| 金額必須     | 0より大きい     | "振込金額を正しく入力してください"             |
| 振込上限     | ≤ ¥1,000,000    | "1回の振込上限（¥1,000,000）を超えています"    |
| 残高チェック | 金額 ≤ 残高     | "残高不足です"                                 |
| 自己振込禁止 | 振込先 ≠ 自口座 | "自分の口座には振込できません"                 |

## 5. 承認フロー

- 条件: 金額 ≥ ¥100,000 かつ 役割 = 一般顧客
- 動作: 振込実行せず、「承認待ち」状態に追加
- メッセージ: "¥100,000以上の振込は管理者承認が必要です"

## 6. 成功時の処理

- 残高から振込金額を減算
- 取引履歴に追加（種別: 振込、金額: マイナス値）
- 画面の残高表示を更新
- 成功メッセージ表示（3秒後に自動消去）
- 監査ログ記録

## 7. 監査ログ

### 7.1 記録対象イベント

| レベル  | イベント例                 |
| ------- | -------------------------- |
| warning | 承認待ち振込、振込上限超過 |
| info    | 振込実行                   |

## 8. テストデータ

| ユーザーID | 役割     | 残高       | 口座番号   |
| ---------- | -------- | ---------- | ---------- |
| customer01 | customer | ¥1,500,000 | 1234567890 |
| admin01    | admin    | ¥5,000,000 | 9876543210 |
| auditor01  | auditor  | ¥2,000,000 | 5555555555 |

## 9. テスト観点

- 正常系: 条件を満たす振込成功
- 異常系: 口座番号不正、金額不正、残高不足、自己振込
- 境界値: 1円、100,000円、1,000,000円、1,000,001円
- セキュリティ/業務ルール: 高額振込の承認待ち
